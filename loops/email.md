# Email/Communication Loop

Paste this into your AI tool. Give it a brief description of what you want to communicate. It will draft, review, and polish automatically.

## Loop Specification

### GOAL
Produce a clear, appropriate, and polished message that achieves the communication goal.

### PLAN
1. Understand the context: who is the recipient? What's the relationship?
2. Clarify the goal: what do you want the recipient to do/feel/know?
3. Draft the message with appropriate tone and length.
4. Self-review: check clarity, tone, politeness, completeness.
5. Edit: remove filler, fix awkward phrasing.
6. Final check: would YOU want to receive this message?

### EXECUTE
- Draft in full before editing.
- After each revision, highlight what changed.
- Consider cultural context (formal vs informal, direct vs indirect).

### VERIFY Checklist
- [ ] Purpose is clear within the first 2 sentences
- [ ] Tone matches the relationship (formal for professors, casual for peers)
- [ ] No ambiguity — recipient knows exactly what's being asked/said
- [ ] No typos or grammar errors
- [ ] Appropriate length (not too short to be rude, not too long to bore)
- [ ] Call to action is explicit (if applicable)

### FIX Rules
- If the message is > 200 words, consider if it can be shorter.
- Remove all "I hope this email finds you well" type filler.
- Replace vague requests with specific ones.
- If unsure about tone, err on the side of more formal.

### STOP Conditions
```
STOP when:
- All checklist items pass
- Message is concise and clear
- Tone is appropriate

STOP and ASK when:
- Relationship context is unclear
- Sensitive topic (conflict, complaint, negotiation)
- Cultural norms are uncertain
- Multiple valid approaches (formal vs informal)
```

### Output Format
```
## Message Draft

### Context
- Recipient: [who]
- Relationship: [professor / colleague / student / ...]
- Goal: [what you want to achieve]

### Draft
[The message]

### Self-Review Notes
- Tone: [formal/casual, appropriate?]
- Length: [X words, too long/short?]
- Clarity: [any ambiguity?]
```
