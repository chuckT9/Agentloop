# AgentLoop

> **别再单次提问了。用循环。** 设计自主运行的 AI 工作流 — 自动规划、执行、验证、修复 — 不需要写代码。

你打一段话。AI 给你一个回答。你检查，发现不对。你换个说法再问。还是不对。你放弃了。

**AgentLoop 解决这个问题。** 不是单次提问，而是给 AI 一个**循环** — 一个"规划 → 执行 → 验证 → 修复"的结构化流程，直到任务真正完成。

## 快速开始（2 步）

### 第 1 步：选一个循环

浏览 `loops/` 文件夹，选一个匹配你任务的：
- `code-review.md` — 代码审查
- `debugging.md` — 调试修复
- `data-analysis.md` — 数据分析
- `writing.md` — 写作编辑
- `research.md` — 研究调研
- `email.md` — 邮件撰写

### 第 2 步：粘贴

**ChatGPT：** 在对话开始时粘贴循环内容。

**Claude：** 粘贴到项目指令或对话开头。

**Cursor / Copilot：** 添加到项目的 `.cursorrules` 或系统指令。

**Hermes Agent：** 放到 skills 文件夹或粘贴到 AGENTS.md。

然后给 AI 一个任务。它会自动按循环执行。

## 设计自己的循环

每个循环包含 6 个要素：

1. **目标** — 最终要达成什么？
2. **规划** — 分几步？
3. **执行** — 每步做完，展示过程
4. **验证** — 可观测的检查清单
5. **修复** — 验证不通过就修，修完再验
6. **停止** — 什么时候结束（什么时候问人）

完整指南见 `docs/LOOP-DESIGN.md`。

## 黄金法则

1. **一次一步。** 完成 → 验证 → 下一步。
2. **展示过程。** 每轮迭代报告改了什么。
3. **错误大声说。** 不要藏错误。
4. **两次出局。** 同一个错误出现两次 = 停下来问人。
5. **追踪状态。** 知道自己在循环的哪一步。
6. **人工检查点。** 重要节点暂停等人确认。

## 真实案例

### 案例 1：研究生写论文章节
一个研究生要写方法论章节。她没有让 AI "帮我写"，而是粘贴了**写作循环**。AI 逐节写作，自动检查学术语气、图表引用、逻辑连贯性，所有检查通过才停止。结果：2 轮迭代出定稿，而不是反复重新提问 5 次。

### 案例 2：开发者调试线上报错
一个开发者收到一个 traceback。他粘贴了**调试循环**。AI 读错误信息、追踪执行路径、列出 3 个假设、逐一验证、应用修复、确认错误消失。1 轮循环搞定根因定位。

### 案例 3：研究者分析实验数据
一个研究者有一批实验数据。她粘贴了**数据分析循环**。AI 自动做数据画像、清洗、探索性分析、生成可视化图表，最后产出一份带可操作建议的分析报告。一个循环搞定，不用反复提问。

### 案例 4：写一封专业的邮件
你需要给导师发一封合作邮件。粘贴**邮件循环**，AI 起草、检查语气（够正式吗？）、去掉废话、产出一封简洁清晰的邮件。再也不用对着空白发件箱发呆。

完整循环模板见 `examples/` 文件夹。
